5) In BOTH actual AR engine files (S1 and S2), make two small engine-level changes.

A. Replace the camera request:

with:

const arProfile = window.AIQuestARRuntime
  ? window.AIQuestARRuntime.getCameraConstraints(true)
  : true;
const stream = await navigator.mediaDevices.getUserMedia({ video: arProfile, audio: false });
video.srcObject = stream;
if(window.AIQuestARRuntime){
  window.AIQuestARRuntime.registerVideo(video);
}

B. Immediately before any MediaPipe / detector inference call such as:

insert:

if(window.AIQuestARRuntime && !window.AIQuestARRuntime.shouldProcessFrame()){
  requestAnimationFrame(loop);
  return;
}

C. After creating the detector, video stream, or animation loop, register cleanup:

if(window.AIQuestARRuntime){
  window.AIQuestARRuntime.registerCleanup(() => {
    try { cancelAnimationFrame(rafId); } catch(_) {}
    try { stream && stream.getTracks().forEach(t => t.stop()); } catch(_) {}
    try { detector && detector.close && detector.close(); } catch(_) {}
    try { video && (video.srcObject = null); } catch(_) {}
  });
}

- S1 engine loads once only when the AR URL requests S1.
- S2 engine loads once only when the AR URL requests S2.
- Ordinary sessions have no MediaPipe/camera overhead.
- Camera is released when learners leave AR, hide the tab, or close the page.
- AR inference is capped at 18 fps on smaller screens and 24 fps otherwise.
