HOW TO PLAY W2

นำทีมจากหลักฐานไปสู่การเรียนรู้

  1. Empathize — อ่าน User Signal โดยไม่ด่วนสรุปแทนผู้ใช้
  2. Define — เขียนปัญหาที่โฟกัสคนและความต้องการ ไม่กระโดดไปหา solution
  3. Ideate — เลือกแนวคิดที่ทดสอบสมมติฐานสำคัญก่อน
  4. Prototype — ทำต้นแบบเท่าที่จำเป็นเพื่อให้ผู้ใช้ลองทำ task จริง
  5. Test & Iterate — สังเกตพฤติกรรม แล้วกลับไปแก้ phase ที่ต้นเหตุ

ผ่าน 1★ เมื่อทำครบ 5 phase และได้ Sprint อย่างน้อย 50 คะแนน
