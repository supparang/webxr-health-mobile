# UX Quest — W2 to B5 Release Standard

## Campaign path

`W2 → W3 → B1 → W4 → W5 → W6 → B2 → W7 → B3 → W9 → W10 → W11 → B4 → W12 → W13 → W14 → B5`

- W8 is represented by **B3: UX Blueprint Gauntlet**.
- W15 is represented by **B5: UX Launch Defense**.
- A learner needs **2 stars** to unlock the following node.

## Shared game rule

Every playable node uses:

1. Three context-rich casefiles selected with recent-case protection.
2. Four decision options: one evidence-led answer plus three near-miss alternatives.
3. Per-question answer-position shuffling.
4. Reason Check after a correct decision; 3 stars requires decision accuracy and verified reasoning.
5. Hint cost and combo scoring, so a fast guess does not create high mastery.
6. Result integrity: pass, badge, unlock and handoff must agree.
7. Studio Transfer artifact saved locally for use in the worksheet, prototype or final case study.

## Boss escalation

- **B1/B2:** synthesis missions using the foundation/flow cases already used in the campaign.
- **B3/B4/B5:** after the core boss case, the game adds Boss Signal decisions taken from a second rotating casefile. This forces transfer, not memorization.

## Teacher acceptance test

Open a route with `?qa=1&preview=1`.

Teacher Preview is local only:

- prerequisite bypass applies only to that tab;
- classroom receiver delivery is disabled;
- normal student routes are unchanged.

For each mission test:

1. **Random Path** — select plausible but incomplete options; it must not accidentally clear 2 stars.
2. **Golden Path** — use the evidence-led choice and matching Reason Check; it must clear 2 stars and show the next handoff.
3. **Replay Path** — replay and confirm a different case or different answer positions appear.
4. **Studio Path** — confirm the mission-specific Studio Artifact is visible after results and can be stored locally.
