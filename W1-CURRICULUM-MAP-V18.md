## Official Week 1 lenses

| Lens | Learner diagnostic question | UI focus |
|---|---|---|
| Clarity | ผู้ใช้เข้าใจว่าจะเริ่มงานหรือกดอะไรต่อหรือไม่? | labels, menu names, entry points |
| Hierarchy | งานหลักและข้อมูลสำคัญเด่นก่อนหรือไม่? | information order, primary action |
| Cognitive Load | ผู้ใช้ต้องคิด/จำ/เปรียบเทียบเกินงานปัจจุบันหรือไม่? | density, choice load, progressive disclosure |
| Consistency | ผู้ใช้ต้องเรียนรู้คำหรือพฤติกรรมของระบบใหม่ซ้ำหรือไม่? | terminology, states, predictable behaviour |
| Feedback | หลังผู้ใช้กระทำ ระบบบอกสถานะและก้าวถัดไปหรือไม่? | confirmation, status, next action |
| Recovery | หากผู้ใช้พลาด ระบบช่วยแก้และกลับเข้าสู่งานได้หรือไม่? | error prevention, undo, repair path |

## Required learner sentence

> ผู้ใช้ ___ ไม่สามารถ/ไม่มั่นใจที่จะ ___ เพราะ ___ ส่งผลให้ ___

## In-game evidence of learning

1. Identifies user, task and context.
2. Separates UI symptom from UX impact.
3. Diagnoses one friction using one official Week 1 lens.
4. Proposes one evidence-backed improvement, then reads User Test results before deciding what happens next.

Accessibility, privacy and trust remain authentic constraints inside the cases. In V18 they are not treated as extra Week 1 lenses; they are analysed through the six official lenses above.
